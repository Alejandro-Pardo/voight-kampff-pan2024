## 🎯 Task Description

The Voight-Kampff shared task challenges participants to determine whether a given text was written by a **human** or generated by an **LLM**. Given a pair of texts, the system must predict which one is human-written and output a confidence score.

See the [official PAN 2024 page](https://pan.webis.de/clef24/pan24-web/generated-content-analysis.html) for the full task description and datasets.

---

## 🔬 Approach

Our pipeline consists of three stages:

<details>
<summary><strong>Stage 1 — Data Preprocessing & Augmentation</strong></summary>

- Text documents are split into **token-based chunks** (~64 tokens each) using the `Alibaba-NLP/gte-base-en-v1.5` tokenizer.
- **Data augmentation** is performed by rewriting texts with local LLMs (`llama 3.2 1b`, `qwen 2.5 1b`, `gemma 2 2b`) via [Ollama](https://ollama.com/).
- **Noise injection** using `pyleetspeak` to simulate the leetspeak perturbations introduced by the competition organizers.

</details>

<details>
<summary><strong>Stage 2 — Contrastive Learning on Sentence Embeddings</strong></summary>

- We fine-tune **Alibaba-NLP/gte-base-en-v1.5** using **triplet loss** (contrastive learning), training the model to pull human text embeddings closer together and push AI-generated embeddings apart.
- Only the last **2 of 12** transformer layers are unfrozen during training.
- Models were trained for **3, 10, and 20 epochs** on [Kaggle](https://www.kaggle.com/).

</details>

<details>
<summary><strong>Stage 3 — SVM Classification with Calibrated Probabilities</strong></summary>

- A **Linear SVM** is trained on the fine-tuned embeddings to classify individual chunks as human or AI.
- The SVM is calibrated using `CalibratedClassifierCV` to output probabilities.
- For full-text classification, chunk-level probabilities are averaged and combined using a scoring formula inspired by the [BaselineAvengers](https://pan.webis.de/clef24/pan24-web/generated-content-analysis.html) team.

</details>

## 📊 Results

Our best model (20 epochs) achieves the following scores on our test set:

| Metric | Score |
|:---|:---:|
| ROC-AUC | 0.993 |
| Brier | 0.924 |
| C@1 | 0.951 |
| F1 | 0.951 |
| F0.5u | 0.953 |
| **Mean** | **0.955** ✨ |

### Comparison with Competition Leaderboard

| # | Team | ROC-AUC | Brier | C@1 | F1 | F0.5u | Mean |
|:---:|:---|:---:|:---:|:---:|:---:|:---:|:---:|
| 1 | marsan | 0.961 | 0.928 | 0.912 | 0.884 | 0.932 | 0.924 |
| 2 | you-shun-you-de | 0.931 | 0.926 | 0.928 | 0.905 | 0.913 | 0.921 |
| 3 | baselineavengers | 0.925 | 0.869 | 0.882 | 0.875 | 0.869 | 0.886 |
| — | Baseline (Falcon-7B) | 0.751 | 0.780 | 0.734 | 0.720 | 0.720 | 0.741 |

> ⚠️ **Note:** Our results are not directly comparable since we evaluate on our own test split rather than the official competition test set.

We also validated robustness on external data from [Kaggle AI vs Human Text](https://www.kaggle.com/datasets/shanegerami/ai-vs-human-text/data), achieving a mean score of **0.891**.

## 🚀 Getting Started

### Installation

```bash
pip install -r requirements.txt
```

### Usage

The entire pipeline is contained in `transformer.ipynb`. Open it in Jupyter and run the cells sequentially:

| Step | Description |
|:---:|:---|
| 1 | **Data Augmentation** *(optional)* — Generate additional training data using Ollama |
| 2 | **Data Preprocessing** — Load and chunk the text data |
| 3 | **Embedding Model Training** — Fine-tune the sentence transformer with triplet loss |
| 4 | **SVM Training** — Train and calibrate the classifier on embeddings |
| 5 | **Evaluation** — Run the full evaluation pipeline |

> 💡 Training was done on Kaggle with free GPU. You can use the pretrained model weights above to skip the training step.
