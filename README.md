# Voight-Kampff: Generative AI Authorship Verification

Our proposal for the [Voight-Kampff Generative AI Shared Task (PAN @ CLEF 2024)](https://pan.webis.de/clef24/pan24-web/generated-content-analysis.html), developed by **Alejandro Pardo** and **Pedro Amaya** as part of the NLP course in the Master's program *Aprendizaje Automatico y Datos Masivos* at **Universidad Politecnica de Madrid** (2024-2025).

## Task Description

The Voight-Kampff shared task challenges participants to determine whether a given text was written by a human or generated by an LLM. Given a pair of texts, the system must predict which one is human-written and provide a confidence score.

For the full task description and datasets, see the [official PAN 2024 page](https://pan.webis.de/clef24/pan24-web/generated-content-analysis.html).

## Approach

Our pipeline consists of three stages:

### 1. Data Preprocessing & Augmentation
- Text documents are split into **token-based chunks** (~64 tokens each) using the `Alibaba-NLP/gte-base-en-v1.5` tokenizer.
- **Data augmentation** is performed by rewriting texts with local LLMs (`llama 3.2 1b`, `qwen 2.5 1b`, `gemma 2 2b`) via Ollama.
- **Noise injection** using `pyleetspeak` to simulate the leetspeak perturbations introduced by the competition organizers.

### 2. Contrastive Learning on Sentence Embeddings
- We fine-tune **Alibaba-NLP/gte-base-en-v1.5** using **triplet loss** (contrastive learning), training the model to pull human text embeddings closer together and push AI-generated text embeddings apart.
- Only the last 2 of 12 transformer layers are unfrozen during training.
- Models were trained for 3, 10, and 20 epochs on [Kaggle](https://www.kaggle.com/).

### 3. SVM Classification with Calibrated Probabilities
- A **Linear SVM** is trained on the fine-tuned embeddings to classify individual chunks as human or AI.
- The SVM is calibrated using `CalibratedClassifierCV` to output probabilities.
- For full-text classification, chunk-level probabilities are averaged and combined using a scoring formula inspired by the [BaselineAvengers](https://pan.webis.de/clef24/pan24-web/generated-content-analysis.html) team.

## Results

Our best model (20 epochs) achieves the following scores on our test set:

| Metric  | Score |
|---------|-------|
| ROC-AUC | 0.993 |
| Brier   | 0.924 |
| C@1     | 0.951 |
| F1      | 0.951 |
| F0.5u   | 0.953 |
| **Mean**| **0.955** |

## Getting Started

### Installation

```bash
pip install -r requirements.txt
```

### Usage

The entire pipeline is contained in `transformer.ipynb`. Open it in Jupyter and run the cells sequentially:

1. **Data Augmentation** (optional) - Generate additional training data using Ollama
2. **Data Preprocessing** - Load and chunk the text data
3. **Embedding Model Training** - Fine-tune the sentence transformer with triplet loss
4. **SVM Training** - Train and calibrate the classifier on embeddings
5. **Evaluation** - Run the full evaluation pipeline

> Training was done on Kaggle with free GPU. You can use the pretrained model weights to skip the training step.
